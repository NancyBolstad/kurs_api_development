Important To Know

To use React we need to import it as a library. However, we no longer write JS, but rather JSX. JSX is a format in which HTML and JS can be easily mixed together. That is the reason why all the files now ends in .jsx instead of .js.

The browser has no clue of JSX, and all these files need to be transformed into JS before they can be used in a browser. To do that, we need to instruct WebPack to use Babel with React preset when building the bundle.js file. Look at the configuration files .babelrc and webpack.config.js to see what settings are needed to achieve it. Meditate on the lack of comments there, and how it could have been helpful to have them directly there where the different configuration settings are added. Sorry, not going to write abominations like "//": "...".

When you look at tutorials and examples online, you might notice that often for JSX the file extension is .js instead of .jsx. This does simplify some configuration settings (e.g., in WebPack), but I do not like it. JSX is not JavaScript, and using .js would be confusing in my opinion.
